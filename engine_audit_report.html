适用场景 · 可视化报告落地路径

SCENE · 01

一次性数据报告

数据 → Python 处理 → 生成单页 HTML（SVG/ECharts）→ OpenPreview 验证。最稳路径，10 分钟出成品。

SCENE · 02

模板化周报

把报告骨架、配色、章节封装为 Skill，触发即按 SOP 输出，保证多期一致性。

SCENE · 03

定时自动出报

Schedule(cron) + Skill + 数据源 MCP，每日/每周自动跑。受限于"最低每小时一次"的频率上限。

SCENE · 04

多源复杂报告

Task subagent 并行：采集 / 分析 / 渲染各自独立，主会话仅汇总，规避长上下文压力。

SCENE · 05

图文混排报告

text_to_image 生成封面/插图 + HTML 排版正文。注意图片为固定 1832px，需 CSS 适配。

SCENE · 06

交互式仪表盘

HTML + 内联 SVG/CSS 即可实现筛选、Tab、动效；如需 ECharts 需确保目标环境可联网加载 CDN。

已知局限 · 落地前必读

!
无原生 PDF/PPT 导出。需通过 Puppeteer/jsPDF 等代码生成，或浏览器打印另存。
!
图片接口走 302 重定向。下载需 -L 跟随；返回 JPEG 但请求参数名是 .png，扩展名需手动校正。
!
非所见即所得。"写代码 → 预览"流程，非拖拽式编辑器，对非技术用户有门槛。
!
定时任务频率受限。仅支持 cron 5 字段，最低每小时 1 次，不支持秒级与"每月第 N 个周五"。
!
无内置数据库/BI 直连。需自配 MCP Server 或通过 RunCommand 跑脚本对接。
!
长报告上下文压力。超长文档建议拆给 subagent，避免主会话 token 爆满。
!
质量依赖提示词。报告结构、指标口径需明确给出，否则易跑偏。
!
沙箱无外网 CDN 保证。可视化尽量内联 SVG/CSS，避免依赖外部图表库 CDN。
