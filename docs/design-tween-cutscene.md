# 设计稿：补间 + 漫插过场系统

状态：合同草案（粒子系统落地后开工）。定位：演出层第二块——给游戏配片头/片尾/章节插页，素材形态对齐 AI 产能（静态插画，不做视频解码）。

## 一、补间（公共零件，先做）

数值从 A 平滑变到 B 的确定性插值原语，淡入淡出/镜头推拉/UI 弹出/帧动画过渡共用。

- **归属**：模拟层（vitric-sim 或 rules 动作），不是渲染层——补间改的是组件字段（真相源），渲染只管画。进状态哈希、进存档，snapshot/restore 往返一致。
- **形态**：组件 `Tween`（或规则动作 `tween`，实现者按现有风格定）：目标实体+字段路径、起值/终值、起始 tick、时长 tick、缓动曲线。
- **缓动曲线**：固定枚举 linear / ease-in / ease-out / ease-in-out / ease-out-back（过冲回弹）。全部解析式，禁浮点累加积分——第 T tick 的值 = f(progress)，纯函数。
- **端点钉死（两端都钉）**：起跑 tick（elapsed=0）写**精确起始值**——曲线在 p=0 数学上为 0 但浮点求值会留 ~2e-16 尾巴，`apply()` 里 `p==0.0` 直接返回 0.0，端点不交给浮点碰运气（OutBack 尤其会）。完成 tick 写**精确终值**——不走曲线公式，调用方直接落 `to`。两端都是字节级精确，中途才走解析式。
- **完成语义**：到期那一 tick 把字段精确写成终值（不留浮点尾巴），并发完成事件（带补间 id），随后补间自动移除。规则可以听完成事件接龙下一段。
- **冲突**：同一实体同一字段同时只允许一个活跃补间，后来者顶掉前者（显式语义，不报错），文档写明。

## 二、漫插过场（cutscene）

一种特殊场景：一条声明式时间轴驱动"出图→镜头缓移→字幕→切页→淡出"。

- **数据形态**：`cutscenes/<名字>.json`，时间轴 = 有序条目数组，每条 `{ at: <tick>, do: <动作> }`。动作集（v1 固定，不图灵完备，与规则动作风格对齐）：
  - `show`：显示一张插画（实体+贴图），可带入场方式 fade/cut；
  - `pan`：镜头从矩形 A 缓移缩放到矩形 B（Ken Burns 效果），内部即补间；
  - `text`：字幕逐字浮现（速度=字/秒，确定性按 tick 折算）；
  - `sound`：音效/配乐点；
  - `wait_input`：停住等任意确认键（输入走录像通道，回放安全）；
  - `end`：发过场结束事件并按清单跳下一场景。
- **跳过**：全程可跳过是玩家默认期待（release-checklist 已有此项）——专用跳过键直接快进到 `end` 条目执行，跳过本身是一次输入，录像里有记录，重放一致。
- **校验**：`vitric check` 验时间轴——贴图引用存在、at 单调不减、end 必须有且仅有一个、动作字段过 schema。错误带路径+修复提示。
- **确定性**：时间轴播放器是普通系统，按场景内 tick 推进；无墙钟、无随机。gate 可以用 must_emit 过场结束事件认证"片头能放完能跳过"。
- **letterbox**（上下黑边）与全屏淡入淡出做成内置开关，不让每个游戏自己拿黑色精灵糊。

## 三、验收（写给未来的 gate）

1. 补间：缓动五曲线单测逐值断言；完成 tick 字段=精确终值；snapshot/restore 中途回退后续播一致。
2. 过场：录像里含"看完整段"与"中途跳过"两条路径，重放哈希一致；check 对坏时间轴报路径错误。
3. 首个用户：《回响》开场（孩子走进长廊点亮第一盏灯，3-4 张 AI 插画）+ 通关结局页，进它的 gates 块。

## 四、性能预算（硬要求）

演出层不许成为帧率刺客。前车之鉴：桌宠 2400 帧不压缩吃掉 8.5G 显存、阴影最坏开销让 4090 卡帧。本系统从第一天起守这几条：

1. **空场零成本**：没有活跃补间 / 没有过场在播时，每 tick 走 early-return，零分配零遍历（补间侧 `query(&["Tween"])` 空集即返回，已落地；过场播放器同理先判活跃再干活）。
2. **补间是 O(活跃补间数)**，不是 O(实体数)——只遍历挂了 `Tween` 的实体，且补间到期自动移除不堆积。活跃补间数天生是个位数级，每 tick 那点小 Vec 分配可接受，但不许在循环里克隆大对象 / 反复查全表。
3. **打字机文字：版面只算一次**。逐字浮现绝不能每 tick 把整段文字重新排版/测宽——那是按帧数 × 字数的浪费。版面（每个字的位置）在文字首次出现时算一次并缓存（缓存在渲染层、按当前文字 id memo，文字不变不重算），每 tick 只改"画到第几个字"，可见字数 = elapsed 的纯函数。
4. **全屏叠加层是单个四边形**：letterbox 黑边（两个矩形）、全屏淡入淡出（一个带 alpha 的全屏四边形）、过场插画（一张精灵）——全部在现有渲染 pass 里画，**禁止新建离屏 render target / framebuffer / 每帧上传纹理**。淡出就是改一个四边形的 alpha，不是叠一层离屏缓冲。
5. **渲染热路径零每帧堆分配**：复用缓冲，与粒子路径一致（粒子已做到顶点流复用）。
6. **镜头 pan = 解析式补间**，无累加积分；过场播放器状态最小化（当前条目下标 + 起始 tick），快照便宜。
7. **CPU/GPU 两条路径同一份数据**：不许为了 GPU 好看在 GPU 侧多算一遍版面/多铺一层粒子，位置/数量/可见字数两路严格一致（粒子已立此规矩，过场照搬）。

验收时这几条要有测试或 profile 数据兜底，不是口头承诺：空场 early-return 有单测；打字机版面重算次数可断言（同一段文字播 N tick，排版只发生 1 次）。

## 五、刻意不做

- 视频解码/播放（坑大收益低，帧动画即穷人视频）；
- 时间轴里嵌脚本（保持声明式，复杂演出归 rules/scripts 既有通道）；
- 骨骼/网格变形动画（帧动画优先，见路线图）。
