# 设计稿：帧动画深度优化

状态：立项合同（导演定稿）。定位：**帧动画是 Vitric 的一等公民**。AI 出动画的现实路径 = 生成视频/序列图 → 切帧；骨骼绑定恰是 AI 最不擅长的活。所以引擎把"帧动画进口方便 + 内存极致 + 运行时好用"做到位，比追骨骼动画更对个人开发者的胃口。

**重心是进口流水线 + 极致内存**（用户明确："优先支持帧动画并做好极致优化"）。前车之鉴：PetClaw 桌宠 2400 帧不压缩、RGBA8 全驻留 → 8.5G 显存、4090 卡帧。

## 现状（已有，不重做）

- **播放**：`Anim` 组件（clip/prev/t/done）+ `advance_animations`，按 tick 确定推帧，引擎独占 `Sprite.image` 写权（动画不被别的逻辑打断，换动画只改 `Anim.clip`）。
- **素材**：`vitric assets` 已有 median-cut 统一色板、程序化/AI（豆包 Ark）法线。
- **缺口**：①没有帧进口流水线（`--frames`）②GPU 纹理全是 `Rgba8Unorm`，无压缩（8.5G 同款根源）③没有帧事件 ④没有动画状态机。

## 一、进口流水线 `vitric assets --frames`（阶段 1 核心）

把"AI 生成的一堆帧图"一键变成可用、优化过的动画素材。

- **输入**：序列图（png 序列，按文件名排序）。**视频**：不内置解码器（依赖最小、不写 fallback）；用户先用 ffmpeg 把 mp4 转成 png 序列，或 `--frames` 检测到系统有 ffmpeg 时可选调用（没有就明确提示让用户自己转，不静默失败）。
- **处理流水线**（全部确定性，同输入同产物）：
  1. **相邻帧去重**：逐像素近乎相同的相邻帧只留一张，记录"停留多少帧"（很多 AI 动画有静止段，去重砍磁盘+显存）。
  2. **裁空白边 trim**：每帧裁掉透明边，记录偏移（播放时按偏移摆回原位，视觉不变）。
  3. **打包图集 atlas**：所有帧拼进一张大图（减少 GPU 纹理切换），记录每帧 uv 矩形。
  4. **统一色板**：复用现有 median-cut，整组帧一套色板（和谐 + 为压缩铺路）。
  5. **写动画配置**：帧表（uv + 停留帧数 + trim 偏移）、时长、锚点——让 `advance_animations` 照常确定播放。
- **产物**：图集 png + 动画配置 json（+ 可选压缩纹理，见第二节）。`vitric check` 校验产物：图集存在、帧表合法、配置引用一致，错误带路径。

## 二、极致内存：GPU 压缩纹理（阶段 1 核心）

显存大头是 RGBA8 全驻留。砍它：

- **离线压缩**：图集压成 **BC7（BPTC）**——4-6 倍压缩、质量高。压缩在 assets 流水线离线做（产物落盘），不在运行时实时压。编码器选一个能在本工程编译的方案（纯 Rust 编码器优先；ISPC 绑定若容器编译不过就换），选型写进交接单。
- **GPU 上传**：wgpu 路径支持上传 BC7 压缩纹理（需要 device feature `TEXTURE_COMPRESSION_BC`；不支持的设备**显式报错、不 fallback** 到 RGBA8 静默膨胀——让问题暴露，用户知道这台机器没 BC 支持）。
- **CPU 真相源路径不变**：CPU 是参考渲染，仍用 RGBA8，不需要压缩（它不吃显存）。
- **目标**：几百帧的角色控制在几十 MB 显存（对照 8.5G）。
- **确定性**：压缩纹理只影响 GPU 视觉，**不进模拟状态/哈希**——模拟只认帧索引/clip 名，不认像素。
- **容器无 GPU**：BC7 编码（离线）在容器可做可测；GPU 上传压缩纹理的真机验证在 Windows GPU 机做，代码交叉编译过即可交接。

## 三、运行时（阶段 2）

- **帧事件**：动画配置里带帧事件表（"第 7 帧 emit hit""第 12 帧 emit footstep"）；`advance_animations` 播到该帧 emit 命名事件，规则/脚本接（攻击判定、脚步声）。走现有事件通道，确定性。O(播放中动画数)。
- **轻量动画状态机 AnimGraph**：声明"状态 + 转移条件 → clip"（idle/run/attack 自动切换）。定位：**动画专用的条件→clip 映射语法糖**，比写一堆规则方便，但不与规则打架——它只写 `Anim.clip`，条件求值确定性。若评估下来它和规则重叠太多、收益不抵复杂度，可降级为"文档给规则范式"，在阶段 2 立项时再定。

## 四、性能与确定性（硬要求）

- atlas 减纹理切换；去重砍磁盘+显存；BC7 砍 4-6 倍显存；帧事件 O(播放中动画数)。
- assets 是离线工具，不进运行时确定性循环；但产物必须让运行时**确定播放**（去重的"停留帧数"、trim 偏移都在配置里，播放逐帧可重放）。
- 压缩纹理不进模拟哈希；帧事件走确定性事件通道。

## 五、验收

- **阶段 1**：一个动画 demo——给一组 AI 风格序列图（或程序生成的渐变帧占位），跑 `--frames` 出图集+配置，引擎播放正确（帧序、停留、trim 摆位对）；**显存实测对比**（压缩前 RGBA8 vs BC7，给出倍数）；check 校验产物。
- **阶段 2**：帧事件 must_emit 进 gate；状态机条件切 clip 的录像重放一致。

## 六、刻意不做

- 骨骼/网格变形/IK（AI 不擅长，帧动画优先——这是定位选择）；
- 内置视频解码器（用 ffmpeg 转序列图）；
- 运行时实时压缩（离线压，运行时只上传）；
- 不支持 BC 的设备静默回退 RGBA8（显式报错，不让显存悄悄膨胀）。

## 七、分阶段

- **阶段 1**：进口流水线 `--frames`（去重/trim/atlas/色板/配置）+ GPU BC7 压缩纹理 + 显存实测 demo。**用户最在意的"极致优化"核心。**
- **阶段 2**：帧事件 + 轻量动画状态机。
