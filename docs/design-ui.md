# 设计稿：UI 控件系统

状态：立项合同（导演定稿）。定位：游戏一复杂（菜单/背包/设置/对话框）就需要的控件层。对标 Godot Control / Unity UI，但**声明式、确定性、AI 原生友好、无可视化编辑器**——UI 也是实体 + 组件，进世界状态/哈希/存档/录像，CPU/GPU 双路径，布局是确定性纯计算。

引擎只给通用控件原语；具体界面（主菜单长什么样、背包几列）是游戏项目用这些积木拼的用法，不在引擎里——和序列/演出层同一条原则。

## 一、坐标系：UI 是屏幕空间叠加层

UI 元素锚定到**视口（屏幕）**，坐标是像素 / 0..1 比例，渲染走屏幕空间正交投影，**不经相机变换**——镜头移动/缩放/抖动时 UI 不飘（像 HUD）。这与精灵/粒子走世界空间相反。渲染层加一条屏幕空间 UI 通道，紧接世界渲染之后叠加，**不新建离屏 render target / framebuffer**（只是换一个投影矩阵在同一/紧邻 pass 画）。

## 二、布局模型（锚点 + 容器，不上约束求解器）

两套组合覆盖 2D 游戏 UI 绝大多数需求，都是确定性纯计算：

1. **锚点 Anchor**：任意 UI 元素锚定到父（或视口）的某个点 + 像素偏移 + 尺寸。预设：贴四角/四边、居中、拉伸填满父。覆盖 HUD 角标、居中弹窗、满屏遮罩。
2. **容器 Container 自动排版**：父容器自动摆放子元素，子元素声明拉伸权重（size_flags）。三种：
   - `VBox` 竖排 / `HBox` 横排：按主轴顺序排，带间距 gap、内边距 padding、主轴/交叉轴对齐。
   - `Grid` 网格：固定列数，行高列宽按内容或等分（背包格子）。
   子元素的最终位置/尺寸由容器算出，子元素不自己摆坐标（和灰盒约定一致：尺寸锁定，布局算出来）。

**刻意不做** Flexbox/Yoga 式约束求解器（太重）。布局**脏标记**：只在控件树结构或尺寸变化时重算，不每 tick 全量；空 UI（没有 UI 根）每 tick 零成本 early-return。

## 三、控件集 v1

| 控件 | 职责 | 复用 |
|---|---|---|
| `Panel` | 背景框（纯色 / 精灵 / 九宫格 NinePatch 可选，不拉伸边角） | Sprite |
| `Label` | 文字（含逐字显示） | Text + reveal（已落地） |
| `Button` | 可激活，状态机 + 激活发命名事件 | Panel + Label + 焦点/点击 |
| `Container` | VBox/HBox/Grid 自动排版子元素 | 布局 |

弹出/淡入用 **Tween**（已落地），菜单流程编排用 **Sequence**（已落地）——UI 不重复造动画/时序。

## 四、交互模型（焦点导航 + 点击双轨，v1 不做 hover）

**关键决策**：现在只有点击进录像通道，没有光标位置的每-tick 流；且 hover 是纯鼠标概念，手柄/触屏没有。所以 v1：

- **焦点导航 focus**：一组可聚焦控件，方向输入（`ui-up/down/left/right`）按布局相邻关系移动焦点，确认键（`ui-confirm`）激活当前焦点控件。全离散输入、确定性强、零录像膨胀，**顺带给手柄支持铺路**。焦点环 + 当前焦点存在组件里（进状态/快照）。
- **指针点击 pointer**：复用现有 click 拾取通道（窗口鼠标 `mouse` 事件 / 无头 `input/click` RPC，世界坐标+拾取，已录像可重放）。点击命中按钮 = 激活它。UI 在屏幕空间，需要一条屏幕坐标拾取（点是否落在控件矩形内），与世界拾取并存。
- 两轨都发同一个 **`ui-activate {id, action}`** 命名事件，规则/序列接（切场景、开背包等都走 emit→规则，UI 不内置题材动作）。
- **按钮状态**：`normal` / `focused`（被焦点选中，高亮）/ `pressed`（激活那几 tick 的反馈）/ `disabled`（不可聚焦不响应）。**不做 hover**。按下反馈 = scale + modulate（PetClaw 经验：button_down/up 做缩放+染色，不是只换贴图）。
- hover 留 v2：真要的话把光标位置做成每-tick 录像输入流再说，不拖累 v1。

## 五、主题 Theme

`themes/<名>.json` 或 schema 声明：颜色卷（背景/文字/焦点/禁用）、字号、控件默认边距/内边距、按钮各状态样式。一处定义、全局引用，单个控件可覆盖局部值。换肤 = 换一份 Theme 引用。check 校验控件引用的 Theme 名存在。

## 六、确定性 + 性能（硬要求）

1. **空 UI 零成本**：没有 UI 根时每 tick early-return，零分配零遍历。要有单测。
2. **布局脏标记**：结构/尺寸不变不重算；重算是 O(UI 元素数) 的一趟树遍历，不是每 tick。要有断言：静止 UI 连播 N tick，布局重算 0 次。
3. **屏幕空间叠加无离屏缓冲**：UI pass 复用现有顶点流缓冲，零每帧堆分配（学粒子/文字）。
4. **CPU/GPU 双路径同口径**：UI 在 CPU 真相源路径画得出（矩形+文字），GPU 视觉对齐，布局结果两路读同一份（不在 GPU 侧重算布局）。
5. **焦点/点击判定**：O(可聚焦控件数)，不全表扫描。
6. UI 状态（焦点环、当前焦点、按钮 pressed 计时）全进组件 → 快照/录像安全；按下反馈的 scale/modulate 走 Tween 或 reveal 同款解析式，不浮点累加。

## 七、校验（vitric check）

锚点预设名合法 + 偏移/尺寸数值域、容器类型在 {VBox,HBox,Grid} 内、Grid 列数 ≥1、引用的 Theme/事件/贴图/字体名存在、控件字段过 schema、焦点环引用的实体存在。错误带路径 + VDxxx 码 + 修复提示。

## 八、验收（写给 gate）

1. 一个**菜单 demo**（examples 下新项目或并入 intro）：三个按钮竖排（VBox），焦点导航能上下移焦点、确认键激活；鼠标点击也能激活；激活"开始"按钮 emit 事件 → 规则 load-scene 切走。
2. gate 双录像：一条**走焦点导航**（方向键+确认）、一条**走鼠标点击**，都激活到切场景，must_emit 切场景事件，逐位重放一致。
3. 性能单测：空 UI 零成本、静止 UI 布局重算 0 次。
4. snapshot/restore 中途回退焦点态续播一致。

## 九、分阶段（先 1.1，验收后再 1.2）

- **1.1 布局地基 + 静态控件 + 渲染**：屏幕空间 UI pass（CPU/GPU 双路）、锚点、Container（VBox/HBox/Grid）、Panel、Label（接 Text.reveal）、布局脏标记、check、测试、灰盒 demo（不可交互，先把"摆得对、画得出、镜头不飘、零成本”立住）。
- **1.2 交互 + 主题**：Button 状态机、焦点导航、点击激活（屏幕空间拾取）、`ui-activate` 事件、Theme、菜单 demo + 双录像 gate。

## 十、刻意不做

- 可视化编辑器（我们的"编辑器"是 agent + 控制面，定位差异不是缺陷）；
- 鼠标 hover（v1，见第四节）；
- Flexbox/约束求解器、富文本/HTML 渲染、IME/复杂文本输入、滚动惯性物理（滚动列表 v1 可裁到"无惯性整页/整行滚"）；
- 任何题材专属控件（对话框/技能栏/结算面板）——那些是项目用积木拼的用法。
