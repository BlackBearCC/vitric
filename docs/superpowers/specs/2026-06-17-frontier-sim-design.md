# 《星火》(暂定名) — 设计稿

> 科幻行星拓荒 × 模拟经营 × 会呼吸的活居民。在一颗荒星的一片区域,从孤身落脚建成一座有温度的定居点;来投奔的旅人都是 LLM 驱动的活人——记得你、有作息、会因为你怎么待这片地、待他们而留下或离开。

日期:2026-06-17 · 引擎:Vitric · 代码落点:`games/frontier`

## 先讲清几个词
- **白模**:用最简像素图当占位素材,先把玩法/关卡跑通,美术后期再打磨,不卡开发。
- **活伙伴**:LLM 驱动的非玩家角色,有稳定人设和会成长的日常,区别于写死的传统 NPC。
- **心脏 C**:核心循环——"你为伙伴建设 ↔ 伙伴的反应/习惯又给你继续建的理由",互相喂养。
- **ctx.ask**:引擎脚本里"问大模型拿答案"的统一调用,回答自动录进存档、可逐位回放(已实现)。

## 一、类型与参照
模拟经营 / 建设求生沙盒。参照系:**《缺氧》《RimWorld》那种"建设 + 恶劣星球求生" + 星露谷的暖 + 我们独有的会呼吸的活居民**。本质是**可以一直玩的沙盒**,不是有结局的关卡游戏。

## 二、四根支柱
1. **建设(主)**:核心动词。给一片区域起穹顶、铺电/水/路、造车间、规划居住区,把落脚地一步步建成定居点。玩家大半时间花在这。
2. **生存/挑战(压力)**:星球不友好——电力/氧气/温度/食物要顾,扛宇宙事件(耀斑、陨石雨、异星枯萎、严寒)。生存是"你为什么得不停建设"的理由。**暖式求生**:有真压力、能救回来,不搞死亡螺旋/饿死人;挫折是揪心不是惩罚,暖底不破。
3. **活伙伴(魂)**:见第三节,最大差异化。
4. **心脏 C**:把上面三者拧成一个互喂的循环。

## 三、活伙伴系统(差异化核心)
- **少而深**:同时 3–5 个核心 + 偶尔的流动访客。质量压过数量——你得真懂一个人才知道给他建啥。
- **不同来历**:坠机的、逃难的、流浪的,各有背景脾气;**现生成、一生成即冻住**(人设稳定不漂,保证养得出感情),日常随记忆生长。
- **三层需求模型(一套引擎,三层深度)**:
  - **需求(藏着的引擎)**:每人几条状态(舒适/社交/爱好…),驱动一切,默认不糊数值条。
  - **习惯/观察(看得见)**:需求变成行为——爱好缺了就老往某个空角落蹭,闷了就发呆。你读行为,不是读条。
  - **愿望(偶尔明话)**:某条缺到一定程度,角色口吻里漏一句("好想有个能看书的地方…")当路标。
  - 想深看的玩家可点开某伙伴看更细状态(给爱管理的人留口子);但默认表面是暖的观察 + 愿望。
- **不满意就走,温和有预警**:三层就是走前的预警阶梯(需求没喂饱 → 习惯显冷淡 → 抱怨/提愿望 → 还不管 → 收拾走人)。给足信号和时间,你一回应就留;走是"他觉得这儿不合适、去别处了",不是惩罚式 game over,空位来新人。
- **半自主**:白天的日常用便宜的脚本习惯撑(不烧钱),**LLM 只在有意义的节点开口**(迎你、对你刚改的东西有反应、提愿望、聊天、长出新习惯),走 `ctx.ask`。
- **会帮工**:按本事参与建设/种养,既是经营一环,也让"留住人"有实利。

## 四、经营/建设循环
- **种**:垦地种工程作物,可调环境(光照/大气/养分);管饭,是生存的一块。
- **加工 + 自动化**:原料进反应炉/合成器出成品;**科技树**解锁机器、自动化、新穹顶、新作物——长线进展在此。
- **采集**:矿洞、打捞废船、采异星动植物拿稀有料。
- **卖**:卖给过路商船/前哨站挣 credits,投回扩建。
- **扩建**:盖穹顶、加房间、布置居住区——**经营与伙伴的家在此合一:你建的站,就是他们住的家。**

## 五、游戏曲线(前期 ramp → 无限续航)
前期一条设计好的五幕爬坡,教会 + 钩住,然后接进自我续命的无限循环。原则:**每幕只丢一个新大系统、难度走"紧—松—紧"的呼吸、难度与情绪绑死**。
1. **落脚(孤身求生)**:先别死,撑起氧/电/庇护/口粮。难度偏高但能救。解锁基础建造 + 第一茬作物。钩子:一个求救信号靠近。
2. **第一个伙伴(暖来了)**:第一人投奔,"为人建"开始,需求/习惯/羁绊活起来,心脏 C 上线。生存松一口,换上"留住人"的新压力。
3. **社区成形(经营深化)**:更多不同来历的人来,定居点长大;自动化/加工/贸易/科技树/扩建全铺开;多伙伴有彼此的化学反应。复杂度与宇宙事件升级。
4. **考验(峰值)**:一记重击(大宇宙事件/物资危机/一个伙伴的去留抉择),攒下的系统和人一起扛。难度峰值,前期准备在此兑现。
5. **绿洲(收束 + 通往无限)**:扛过危机,落脚地成兴旺的家;**不是结局,是通往无限玩的门**。

## 六、无限续航引擎(这类游戏的命门)
- **活伙伴 = 无限内容引擎(最大护城河)**:现生成的新旅人源源不断,每个都是没重过的人和故事;来了又走,社交/情感层永不枯竭。星露谷 NPC 有限、处熟到头;我们这层挖不到底。
- **建设/经济/自动化深度无顶**:永远能扩、能优化、能美化。
- **挑战程序化升级**:宇宙事件/威胁随规模涨;区域吃透了还能往外 claim 新地块。
- **开放念想**:没有"通关",只有越来越高的目标(更大更美更高效的定居点、更深的羁绊)。
- **季节/周期**:给永恒的节奏感。

## 七、美术管线
- **像素白模先行,俯视角**:用最简像素占位素材把系统/手感先做扎实,美术不卡进度;建一个**像素素材库**当白模。
- **后期 i2i 美化**:用关卡截图喂图生图,一图探多套视觉方案再挑(Vitric 有无头确定性截图 + 素材管线底子)。最终长相(暖绘本只是候选之一)留到后面探,不在开发期锁死。
- 需要像素图时,用 Windows 那边的 libtv 生。

## 八、引擎落地(Vitric)
- **建设/作物/资源/季节/机器** = 确定性规则 + 数据(引擎主场)。
- **伙伴的脑** = 组件存状态(人设/记忆/计划/心情/羁绊)+ 脚本调 `ctx.ask`;LLM 回复自动录进存档,确定性回放不破。
- **白送**:确定性存档/回放——一段经营、一段友情怎么长起来的能逐位倒带看;快照可分叉。

## 九、范围纪律(YAGNI)
- **做**:一片区域;建设主循环;暖式求生;3–5 活伙伴 + 访客;三层需求 + 温和离开;半自主 + ctx.ask;经营子系统(种/加工/采/卖/扩);五幕前期 + 无限续航;像素白模;存档回放。
- **先不做**:联机;几十号 NPC;战斗;超复杂经济数值;多星球/多区域(先一片);最终精美美术(白模先行)。

## 十、构建阶段(喂给实现计划)
- **P0 地基**:`games/frontier` 工程骨架(vitric.json/schema/场景/规则/脚本)+ 像素白模库 + 俯视角渲染跑通 + `vitric check/run` 绿。
- **P1 建设核心**:网格放置/拆除、基础设施(电/氧/温/路)、资源与背包。
- **P2 生存**:需求(电氧温食)消耗与维持、第一类宇宙事件、暖式失败-恢复。
- **P3 种养**:垦地/作物/环境调节,接进食物。
- **P4 第一个活伙伴**:组件 + ctx.ask 接入(用录制回复跑确定性自测),需求三层最小版,迎你/提愿望/帮一件工。
- **P5 经营深化**:加工/自动化/科技树/贸易雏形。
- **P6 多伙伴 + 离开机制 + 流动访客**。
- **P7 曲线落地**:五幕节奏脚本 + 无限续航的程序化挑战。
- 每个 P 自带白模可玩 + 测试 + 提交。
