# 设计稿：演出层通用原语（补间 + 序列 + 文字显隐）

状态：补间已落地（`vitric-sim` tween.rs），序列与文字显隐待开工。

**定位：引擎只给通用积木，不给任何玩法/题材专属概念。** 对标 Unity Timeline / Godot AnimationPlayer——它们没有"漫画过场类""卡牌动画类"，只有"时间轴 + 补间 + 关键帧属性"这种通用机制；过场、技能演出、UI 动效都是用这套积木拼出来的**用法**，活在游戏项目里，不在引擎里。本系统同此：三块通用原语，互相组合，覆盖一切定时演出。

三块原语的分工：**补间**让一个字段平滑变；**序列**按时间轴依次发动若干通用动作（其中一个动作就是"起一个补间"）；**文字显隐**让文本能被补间/序列驱动着逐字显示。

---

## 一、补间 Tween（已落地）

数值从 A 平滑变到 B 的确定性插值原语。淡入淡出、镜头推拉、UI 弹出、颜色渐变、位移缩放——一切"平滑变化"的底座。

- **归属**：模拟层（`Tween` 组件 + sim 的 `advance_tweens`），改的是组件字段（真相源），渲染只管画。进状态哈希、进存档，snapshot/restore 往返一致。
- **缓动曲线**：固定枚举 linear / ease-in / ease-out / ease-in-out / ease-out-back。全部解析式，禁浮点累加积分——第 T tick 的值 = `from + (to-from)·ease(elapsed/duration)`，纯函数。
- **端点钉死（两端都钉）**：起跑 tick（elapsed=0）写精确起始值（`apply()` 里 `p==0.0` 直接返回 0.0，不留 ~2e-16 浮点尾巴，OutBack 尤其会）；完成 tick 写精确终值（不走曲线公式，直接落 `to`）。中途才走解析式。
- **完成语义**：到期发完成事件（带 id）后自动移除；规则可听完成事件接龙。
- **冲突**：同实体同字段同时只允许一个活跃补间，后来者顶掉前者（显式语义）。

## 二、序列 Sequence（通用时间轴，替代原"过场"）

引擎的通用时间轴原语 = Unity Timeline / Godot AnimationPlayer 的对位物。一条声明式、按 tick 推进的动作轨道，在指定时刻发动**引擎已有的通用动词**——绝不内置任何"显示漫画页""切到下一关"这类场景/题材专属动作。

- **数据形态**：序列定义是项目静态数据（`sequences/<名>.json` 或 schema 里声明），形态 = 有序条目数组 `{ at: <相对 tick>, do: <动作> }`，`at` 相对序列起跑点（同一序列任何时刻起跑都从自己的 t=0 放，区别于规则的绝对 tick 触发）。运行时一个 `Sequence` 组件实例化它，**只持有最小播放状态**（引用哪条序列 + 游标条目下标 + 起跑 tick + 是否在 barrier 等待），静态轨道不进每实例快照，快照便宜。
- **动作集 = 通用动词（v1 固定，不图灵完备、不嵌脚本）**，每个都镜像引擎已有能力，不新造语义：
  - `tween`：起一个补间（复用第一块的 `Tween`）。这一个动作就覆盖了镜头推拉（tween 相机 Position/Zoom）、淡入淡出（tween alpha）、位移/缩放/变色。**序列负责编排，补间负责执行**，零重复。
  - `set`：瞬时设字段值（镜像规则的 set 动作）。
  - `spawn` / `despawn`：生成/销毁实体（镜像）。插画入场 = spawn 一个带 Sprite 的实体。
  - `emit`：发一个事件，让规则/脚本接龙。**序列借此与"场景"解耦**——要切场景就 `emit "to-next"`，由项目里一条规则接住去 load-scene；序列本身不认识"场景""关卡""过场"这些概念。
  - `sound`：播音效/配乐点（镜像音频）。
  - `wait`：barrier——游标停住，直到某命名事件触发 / 某输入到达才继续（输入走录像通道，回放安全）。这是规则不易表达的状态机能力，是序列存在的另一半理由。
- **完成**：序列跑到末尾发完成事件（像补间），后续动作（切场景等）由规则接，不内置。
- **确定性**：序列推进是普通系统，按 tick 走，无墙钟无随机；snapshot/restore 中途回退后续播一致。
- **校验**：`vitric check` 验序列——`at` 单调不减、动作名在固定集合内、动作字段过 schema、引用的实体/贴图/事件名存在。错误带路径 + VDxxx 错误码 + 修复提示。
- **跳过**：序列可被一条 `skip` 输入快进到末尾（把未执行条目里的"终态"动作落定后发完成事件）。跳过是输入，进录像，重放一致。gate 可用 must_emit 完成事件认证"序列能放完、能跳过"。

为什么序列不是规则的重复：规则触发器是**绝对 tick**、无游标、无 barrier；序列给的是**相对起跑的有序轨道 + 等待点**，这正是 Timeline 比裸脚本多出来的那层。两者分工：序列编排"一条定时演出"，规则处理"条件反应"。

## 三、文字显隐 Text.reveal（唯一的具体能力，仍是通用属性）

对标 Godot `Label.visible_characters` / `visible_ratio`：给 `Text` 组件加一个 `reveal` 字段（可见字数或 0..1 比例），渲染只画到该进度。**它本身只是文本渲染的一个通用属性，不绑过场**——逐字打字机 = 用补间把 `Text.reveal` 从 0 tween 到 1；瞬显 = set 成 1；倒着隐 = tween 回 0。谁来驱动它（序列/规则/脚本）都行。

## 四、性能预算（硬要求）

演出层不许成为帧率刺客。前车之鉴：桌宠 2400 帧不压缩吃掉 8.5G 显存、阴影最坏开销让 4090 卡帧。从第一天守这几条，且要测试/profile 兜底，不是口头承诺：

1. **空场零成本**：没有活跃补间 / 没有序列在播时，每 tick early-return，零分配零遍历（补间侧 `query(&["Tween"])` 空集即返回，已落地；序列播放器同理先判活跃）。要有空场 early-return 单测。
2. **补间 / 序列都是 O(活跃数)**，不是 O(实体数)——只遍历挂了对应组件的实体，到期自动移除不堆积。活跃数天生个位数级，每 tick 那点小 Vec 分配可接受，但不许在循环里克隆大对象 / 反复查全表。
3. **文字版面只算一次**：逐字显示绝不能每 tick 把整段文字重排/测宽（按帧数×字数的浪费）。版面（每字位置）首次出现算一次、按当前文字 id memo 缓存在渲染层，文字不变不重算；每 tick 只改"画到第几个字"，可见字数 = `reveal` 的纯函数。要有断言：同一段文字播 N tick，排版只发生 1 次。
4. **全屏叠加层是单个四边形**：黑边（两个矩形）、全屏淡入淡出（一个带 alpha 全屏四边形）、插画（一张精灵）全在现有 render pass 画，**禁止新建离屏 render target / framebuffer / 每帧上传纹理**。淡出 = 改一个四边形的 alpha，不是叠离屏缓冲。
5. **渲染热路径零每帧堆分配**：复用缓冲，与粒子路径一致（粒子已做到顶点流复用）。
6. **序列播放器状态最小化**（游标 + 起跑 tick + barrier 标志），静态轨道不进快照；镜头推拉 = 解析式补间，无累加。
7. **CPU/GPU 两路同一份数据**：不为 GPU 好看多算一遍版面/多铺一层粒子，位置/可见字数两路严格一致。

## 五、验收（写给未来的 gate）

1. 补间：五曲线逐值断言；完成 tick 字段=精确终值；snapshot/restore 中途回退续播一致（已绿）。
2. 序列：录像含"看完整段"与"中途跳过"两条路径，重放哈希逐位一致；must_emit 完成事件被 gate 认证；check 对坏序列（at 乱序/未知动作/引用不存在）报路径错误；空场零成本单测。
3. 文字：reveal 驱动下可见字数 = 纯函数；版面重算=1 次断言。
4. 首个用户：《回响》用这套原语**在项目里**拼出开场（孩子走进长廊点亮第一盏灯）+ 结局页——引擎零过场代码，全是 Sequence + Sprite + Text + Camera + Tween 的组合，进它的 gates 块。

## 六、用法示例：漫画过场 = 拿原语拼出来（活在游戏项目，不在引擎）

证明抽象到位——曾经想做成"引擎功能"的漫画过场，现在是 games/echo 里一条 Sequence：

```
at 0:   spawn 上黑边、下黑边（两个黑矩形精灵）        // letterbox
at 0:   spawn 插画1（Sprite, alpha=0）
at 0:   tween 插画1.alpha 0→1 over 30               // 淡入
at 30:  tween 相机.zoom 1.0→1.15 over 120           // Ken Burns 推镜
at 30:  set 字幕.text "他走进无尽的长廊"; tween 字幕.reveal 0→1 over 60  // 打字机
at 150: wait 玩家确认                                // barrier
at 151: tween 插画1.alpha 1→0 over 20               // 淡出
at 171: emit "intro-done"                           // 规则接住 → load-scene battle-1
```

引擎侧一个"过场"字样都没有。换成技能演出、Boss 登场、结局演职员表，全是同一套积木换条 Sequence——这才是通用引擎。

## 七、刻意不做

- 视频解码/播放（坑大收益低，帧动画即穷人视频）；
- 序列里嵌脚本（保持声明式，复杂逻辑用 `emit` 抛给 rules/scripts 既有通道）；
- 任何题材/玩法专属的高层动作（"显示对话框""发牌""结算面板"）——那些是项目用积木拼的用法；
- 骨骼/网格变形动画（帧动画优先，见路线图）。
