## The problem: new quest gates require wish fulfillment

Task 6 changed the quest gates. The old script drives straight through `companion-moved-in` → step 4, but the new gates require:

| Step | Old gate | New gate |
|------|----------|----------|
| 3→4 | `companion-moved-in` event | `wish-fulfilled` event + `@companion.Need.affinity >= 60` |
| 6→7 | `companion_happy_count >= 1` | `companion_wish_count >= 2` |
| 7→8 | emits `game-won` | emits `settlement-founded` (no `game-won`) |

## Wish system mechanics (confirmed from wish.js + wish.json)

The initial companion "Pip" (entity named `"companion"` in main.json, archetype "话痨技工" → builder) has 3 wishes:
1. **"建造 3 个结构"** — kind: `"build"`, target: 3. Advanced by ANY `built` event (`wish-advance-build` rule). Each build +1.
2. **"建一盏灯"** — kind: `"build-lamp"`, target: 1. Advanced by `built` event with `kind: "lamp"`. **UNFULFILLABLE** — lamps are CRAFTED (emit `crafted`), not BUILT (emit `built`). There is no "lamp" structure kind in the BUILD table. This is a known Minor bug; do NOT try to fulfill it.
3. **"升级 1 个结构"** — kind: `"upgrade"`, target: 1. Advanced by `upgrade-structure` event (Task 7's `upgrade_structure` fn emits this).

Each wish fulfillment: `Need.affinity += 30` (capped at 100), `Colony.companion_wish_count += 1`, emits `wish-fulfilled{companion, wish_desc, entity}`.

Pip's base affinity is 30. One wish fulfillment → 60. This meets the step 3→4 gate (`affinity >= 60`).

## Critical timing constraint for step 3→4

The `quest-step3-done` rule fires on `wish-fulfilled` events, but ONLY when `step == 3`. If the wish is fulfilled BEFORE step reaches 3, the event fires when step is still 2 → gate fails → step never advances.

**The old script builds beacon + 4 plots BEFORE harvesting wheat (which triggers step 2→3).** This means 5 `built` events fire while step is 2. The 3rd build fulfills Pip's "build" wish → `wish-fulfilled` event fires while step==2 → gate fails. Step 3 is reached later (after harvest), but the wish-fulfilled event is already gone.

**Fix**: reorder so the 3rd build happens AFTER step reaches 3:
1. Build beacon → step 2 (1 build, wish progress 1/3)
2. Build 1 plot (2 builds, wish progress 2/3)
3. Plant wheat on the plot
4. Wait for crop to mature, harvest → step 3
5. NOW build 1 more plot (3rd build → wish fulfilled → `wish-fulfilled` event fires while step==3 → gate passes → step 4)
6. Build remaining plots/walls

## Step 6→7: need companion_wish_count >= 2

The first wish (Pip's "build 3 structures") gives count=1. Need one more. Since "build-lamp" is unfulfillable, fulfill the "upgrade" wish:

1. Ensure a tier-1 structure exists (plot/conduit/quarters — the plots built earlier qualify)
2. Press "u" to enter upgrade mode (Task 8 added this keybinding)
3. Click the structure → `upgrade_structure` fn runs → if resources sufficient, upgrades + emits `upgrade-structure` event → `wish-advance-upgrade` rule → Pip's "upgrade" wish fulfilled → count=2
4. Upgrade costs: plot→greenhouse {ore:2, plank:2}, conduit→solar-array {ore:3, plank:1}, quarters→cabin {plank:4, lamp:1}. The seed-start inventory (ore6/plank6/lamp2) covers plot or conduit upgrade.

**Do this BEFORE day 5** (step 7 requires day>=5 + pop>=3 + count>=2, so the upgrade must happen before the day-5 check).
